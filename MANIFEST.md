# MANIFEST

## Stack

Python 3.10+ · anthropic SDK, requests, PyYAML, rich, python-docx, pymupdf · pytest + pytest-mock (119 offline tests) · Claude Code skills (Sonnet orchestrator + Haiku subagents)

## Structure

```
skills/
  research/SKILL.md              # Sonnet orchestrator — 3-tier pipeline coordinator
  research-search/SKILL.md       # Haiku subagent — web search & URL selection
  research-classify/SKILL.md     # Haiku subagent — vault placement, tags, wikilinks

scripts/
  claude_pipe.py                 # Universal Claude API pipe — all analysis scripts call this
  config.py                      # Auto-generated by discover_vault.py (gitignored)
  utils.py                       # Shared helpers: startup_checks, slugify
  discover_vault.py              # One-time setup: scan vault, generate config.py + .env
  ingest.py                      # URL → vault inbox (raw archival via Jina Reader)
  ingest_batch.py                # Batch URL ingestion from file list
  ingest_local.py                # Local file ingestion (.docx/.doc/.pdf/.mp3 → inbox)
  fetch_and_clean.py             # Tier 2: Jina Reader fetch + SHA-256 cache (7-day TTL)
  vault_lint.py                  # Frontmatter validation across vault
  find_broken_links.py           # Unresolved wikilink detection
  find_related.py                # Keyword extraction + vault search for related notes
  synthesize_folder.py           # Folder → MOC via Claude (uses repomix for packaging)
  transcript_processor.py        # Whisper .vtt → structured research notes
  produce_output.py              # Note → downstream format (blog, video script, briefing)
  daily_digest.py                # Summary of today's vault changes → daily note

scripts/prompts/
  README.md                      # Assembly pattern docs
  summarize.txt                  # Generic summarization
  extract_transcript.txt         # Transcript claim extraction
  extract_claims.txt             # Claim extraction
  find_related.txt               # Related note search prompt
  identify_stakeholders.txt      # Stakeholder identification
  synthesize_topic.txt           # Topic synthesis (MOC generation)
  output_formats/                # Downstream templates: web_article, video_script, etc.

tests/
  conftest.py                    # Adds scripts/ to sys.path
  test_*.py                      # One test module per script (119 tests, all offline)

docs/
  handoff-token-efficiency.md    # Token optimization roadmap (3 improvements)
  TAGGING-REFERENCE.md           # Complete tag taxonomy for the pipeline
```

## Key Relationships

- `claude_pipe.py` is the central hub — all analysis scripts import it; enforces prompt-as-user-message pattern and auto-includes vault_rules.txt
- `fetch_and_clean.py` URL validation (SSRF protection) is shared with `ingest.py` — both use the same Jina Reader + cache layer
- `.cache/fetch/` uses SHA-256(url) as key — enables deduplication across ingest.py and the Tier 2 pipeline fetch
- `config.py` is auto-generated by `discover_vault.py` — never edit directly; all scripts depend on its constants
- Skills require `{{placeholder}}` substitution (VAULT_ROOT, SCRIPTS_DIR, PYTHON_PATH, HOME) before use
- Prompt assembly: `{content}\n\n---\n{prompt}\n\n---\n{vault_rules}` — never sent as system messages
